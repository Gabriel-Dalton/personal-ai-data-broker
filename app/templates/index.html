FEATURES

Everything you need to take back control

A complete toolkit for managing what AI services know about you.

Encrypted Data Vault

Store personal data entries by category — contacts, health, finance, preferences. Flag sensitive items for additional protection layers.

Granular Policies

Define per-service rules: which categories to expose, whether sensitive data is included, and request rate limits per hour.

Scoped API Keys

Issue purpose-built keys tied to specific policies. Each AI integration gets the minimum access required — revoke any key in one click.

Complete Audit Trail

Every access attempt is logged — allowed or denied — with timestamps, IP addresses, and request details. Nothing happens without a record.

Self-Hosted

Runs entirely on your machine or private server. SQLite storage, zero cloud dependencies, zero telemetry. Your data never leaves your infrastructure.

REST API + Swagger

Clean REST API with auto-generated interactive docs. Easy to integrate with any AI tool, plugin, or custom workflow.

HOW IT WORKS

Three steps to full control

1

Store your data in the vault

Add entries like contacts, preferences, health info — anything you might want an AI to know. Organize by category. Flag sensitive items. Everything stays in your local SQLite database.

contacts preferences health finance calendar + custom
2

Define access policies

Create a policy for each AI service. Specify which categories it can read, whether sensitive data is included, and a rate limit. Disable or update policies anytime.

policy: "ChatGPT — Read contacts"
categories: [contacts, preferences]
allow_sensitive: false
rate_limit: 60/hr
3

Issue scoped API keys

Generate a unique API key tied to each policy. Hand it to the AI service. The broker enforces your rules on every request — and logs everything in the audit trail.

pdb_k7xR4mP9qZ... Active
